#### 01. Generate a Register Header

Using the following SPI_CR1 register description, prompt your agent to generate a complete C header section (bit-position defines, bit-mask defines, and a bitfield union struct).

```
Register: SPI_CR1 - Control Register 1
Base Offset: 0x00   Reset: 0x00000000

Bit | Field | R/W | Description
----|-------|-----|--------------------------------------------
11  | DFF   | R/W | Data Frame Format (0=8-bit, 1=16-bit)
 9  | SSM   | R/W | Software Slave Management
 8  | SSI   | R/W | Internal Slave Select
 6  | SPE   | R/W | SPI Enable
5:3 | BR    | R/W | Baud Rate Control (000=fPCLK/2 … 111=fPCLK/256)
 2  | MSTR  | R/W | Master Selection (1=Master)
 1  | CPOL  | R/W | Clock Polarity
 0  | CPHA  | R/W | Clock Phase
```

Verify the agent correctly handles the 3-bit `BR` field (mask = `0x38`, not `0x07`).

#### Scenario:

◦ A firmware team is integrating a third-party SPI flash. The reference manual has 40 registers - hand-transcribing all of them would take hours.
◦ The agent must produce output that compiles and matches the datasheet exactly.

**Hint:** The 3-bit `BR` field starts at bit 3. Its mask is `(0x7U << 3)` = `0x38U`.

<details markdown>
<summary>Solution</summary>

```c
/** @defgroup SPI_CR1 Control Register 1 */

#define SPI_CR1_OFFSET      0x00U

/* Bit position defines */
#define SPI_CR1_CPHA_POS    0U
#define SPI_CR1_CPHA_MSK    (1U << SPI_CR1_CPHA_POS)   /*!< Clock Phase */

#define SPI_CR1_CPOL_POS    1U
#define SPI_CR1_CPOL_MSK    (1U << SPI_CR1_CPOL_POS)   /*!< Clock Polarity */

#define SPI_CR1_MSTR_POS    2U
#define SPI_CR1_MSTR_MSK    (1U << SPI_CR1_MSTR_POS)   /*!< Master Selection */

#define SPI_CR1_BR_POS      3U
#define SPI_CR1_BR_MSK      (0x7U << SPI_CR1_BR_POS)   /*!< Baud Rate Control */

#define SPI_CR1_SPE_POS     6U
#define SPI_CR1_SPE_MSK     (1U << SPI_CR1_SPE_POS)    /*!< SPI Enable */

#define SPI_CR1_SSI_POS     8U
#define SPI_CR1_SSI_MSK     (1U << SPI_CR1_SSI_POS)    /*!< Internal Slave Select */

#define SPI_CR1_SSM_POS     9U
#define SPI_CR1_SSM_MSK     (1U << SPI_CR1_SSM_POS)    /*!< Software Slave Management */

#define SPI_CR1_DFF_POS     11U
#define SPI_CR1_DFF_MSK     (1U << SPI_CR1_DFF_POS)    /*!< Data Frame Format */

/* Bitfield struct */
typedef union {
    struct {
        uint32_t CPHA : 1;  /*!< Clock Phase */
        uint32_t CPOL : 1;  /*!< Clock Polarity */
        uint32_t MSTR : 1;  /*!< Master Selection */
        uint32_t BR   : 3;  /*!< Baud Rate Control */
        uint32_t SPE  : 1;  /*!< SPI Enable */
        uint32_t      : 1;  /*!< Reserved */
        uint32_t SSI  : 1;  /*!< Internal Slave Select */
        uint32_t SSM  : 1;  /*!< Software Slave Management */
        uint32_t      : 1;  /*!< Reserved */
        uint32_t DFF  : 1;  /*!< Data Frame Format */
        uint32_t      : 20; /*!< Reserved */
    } b;
    uint32_t w;
} SPI_CR1_TypeDef;
```

**Check:** `BR` mask = `(0x7U << 3)` = `0x00000038U` ✓

#### 02. Validate a Generated Bitfield Struct

The following C bitfield struct was generated by an agent for the `USART_BRR` register. Find all errors and produce the corrected version.

```c
// Agent-generated output (may contain errors):
typedef union {
    struct {
        uint32_t DIV_FRACTION : 4;   /*!< Fraction of USARTDIV */
        uint32_t DIV_MANTISSA : 12;  /*!< Mantissa of USARTDIV */
        uint32_t              : 17;  /*!< Reserved */
    } b;
    uint32_t w;
} USART_BRR_TypeDef;
```

The correct datasheet definition:

- Bits [3:0]: `DIV_FRACTION` (4 bits)
- Bits [15:4]: `DIV_MANTISSA` (12 bits)
- Bits [31:16]: Reserved (16 bits)

#### Scenario:

◦ A firmware engineer reviews agent-generated headers before committing them.
◦ A single off-by-one error in a reserved field can cause the struct to overflow 32 bits, leading to undefined behavior.

**Hint:** The total width of all bitfield members must equal exactly 32 bits.

<details markdown>
<summary>Solution</summary>

**Error found:** Reserved bits are declared as `17` instead of `16`.
`4 + 12 + 17 = 33 bits` - this overflows a 32-bit `uint32_t` union → **undefined behavior** in C.

**Corrected version:**

```c
typedef union {
    struct {
        uint32_t DIV_FRACTION : 4;   /*!< Fraction of USARTDIV: bits [3:0] */
        uint32_t DIV_MANTISSA : 12;  /*!< Mantissa of USARTDIV: bits [15:4] */
        uint32_t              : 16;  /*!< Reserved: bits [31:16] */
    } b;
    uint32_t w;
} USART_BRR_TypeDef;
// Verify: 4 + 12 + 16 = 32 bits ✓
```

</details>

#### 03. Spot the Hallucination

The following C header was generated by an agent for the `ADC_SR` (Status Register) of an STM32F4. One field has a hallucinated bit position that does not match the official datasheet. Identify it.

```c
#define ADC_SR_AWD_POS    0U  /*!< Analog Watchdog Flag */
#define ADC_SR_EOC_POS    1U  /*!< End of Conversion */
#define ADC_SR_JEOC_POS   2U  /*!< Injected Channel End of Conversion */
#define ADC_SR_JSTRT_POS  3U  /*!< Injected Channel Start Flag */
#define ADC_SR_STRT_POS   5U  /*!< Regular Channel Start Flag */
#define ADC_SR_OVR_POS    6U  /*!< Overrun */
```

The real STM32F4 ADC_SR bit positions (from RM0090):

| Bit | Field | Description                        |
| --- | ----- | ---------------------------------- |
| 0   | AWD   | Analog watchdog flag               |
| 1   | EOC   | End of conversion                  |
| 2   | JEOC  | Injected channel end of conversion |
| 3   | JSTRT | Injected channel start flag        |
| 4   | STRT  | Regular channel start flag         |
| 5   | OVR   | Overrun                            |

#### Scenario:

◦ A hallucinated bit position compiles silently but causes wrong register writes at runtime.
◦ Engineers must validate agent output against the official datasheet before use.

**Hint:** Compare each `_POS` value against the table above one by one.

<details markdown>
<summary>Solution</summary>

**Hallucinated fields:**

- `ADC_SR_STRT_POS` is set to `5U` in the agent output, but the real position is **4U**
- `ADC_SR_OVR_POS` is set to `6U` in the agent output, but the real position is **5U**

The agent shifted `STRT` and `OVR` up by one position - a systematic off-by-one error, likely caused by the absence of a gap at bit 4 in the training data.

**Corrected values:**

```c
#define ADC_SR_STRT_POS   4U  /*!< Regular Channel Start Flag */
#define ADC_SR_OVR_POS    5U  /*!< Overrun */
```

**Lesson:** Always cross-reference agent-generated bit positions against the official RM before committing to a project.

</details>

#### 04. Chain Register Map to Initialization Code

Using the SPI_CR1 header you generated in Task 01, prompt the agent to generate a complete `SPI1_MasterInit()` function that:

1. Enables the SPI1 clock via `RCC->APB2ENR` (bit 12)
2. Configures SPI1 in Master mode at `fPCLK/8` (BR=010), 8-bit data, CPOL=0, CPHA=0, SSM=1, SSI=1
3. Enables the SPI peripheral (SPE bit)

Verify the `BR` field is set using the mask define, not a magic number.

#### Scenario:

◦ The register-map agent and the initialization-code agent are chained: the first agent's output is the second agent's context.
◦ The initialization code must use the `#define` names from the header - never raw hex constants.

**Hint:** `BR=010` → value `2U`, written as `(2U << SPI_CR1_BR_POS)`.

<details markdown>
<summary>Solution</summary>

```c
#include "spi_cr1.h"
#include <stdint.h>

#define RCC_BASE        0x40023800UL
#define SPI1_BASE       0x40013000UL
#define RCC_APB2ENR_SPI1EN  (1U << 12)

typedef struct { volatile uint32_t APB2ENR; } RCC_TypeDef;
typedef struct { volatile uint32_t CR1; } SPI_TypeDef;

#define RCC   ((RCC_TypeDef *)RCC_BASE)
#define SPI1  ((SPI_TypeDef *)SPI1_BASE)

void SPI1_MasterInit(void)
{
    /* 1. Enable SPI1 peripheral clock */
    RCC->APB2ENR |= RCC_APB2ENR_SPI1EN;

    /* 2. Configure SPI1:
     *    MSTR=1  → Master mode
     *    BR=010  → fPCLK/8
     *    DFF=0   → 8-bit data frame
     *    CPOL=0  → clock idle low
     *    CPHA=0  → first clock edge captures data
     *    SSM=1   → software slave management
     *    SSI=1   → internal slave select high
     */
    SPI1->CR1 = (SPI_CR1_MSTR_MSK)
              | (2U << SPI_CR1_BR_POS)   /* BR=010 → /8 */
              | (SPI_CR1_SSM_MSK)
              | (SPI_CR1_SSI_MSK);
    /* DFF, CPOL, CPHA remain 0 (reset values) */

    /* 3. Enable SPI */
    SPI1->CR1 |= SPI_CR1_SPE_MSK;
}
```

</details>

#### 05. Automate Multi-Register Generation

Extend the Python automation script from Lab 003 Step 4 to:

1. Accept a directory of `.txt` register description files (one register per file)
2. Generate a header for each file
3. Combine all headers into a single `peripherals.h` with include guards
4. Print a summary: how many registers were processed, how many fields total

#### Scenario:

◦ A real MCU project has 50+ registers across 10 peripherals.
◦ Batch processing replaces a full day of manual work with a 10-second script run.

**Hint:** Use `pathlib.Path.glob("*.txt")` to iterate input files; accumulate content with string join.

<details markdown>
<summary>Solution</summary>

```python
import openai
import pathlib
import re

SYSTEM_PROMPT = """
You are a firmware code generation agent.
Generate a C header section for the following register description.
Use STM32 HAL naming conventions.
Include: offset #define, bit-position #defines, bit-mask #defines, bitfield union struct.
Return only C code, no markdown fences.
"""

client = openai.OpenAI()

def generate_header(register_text: str) -> str:
    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[
            {"role": "system", "content": SYSTEM_PROMPT},
            {"role": "user", "content": register_text}
        ]
    )
    return response.choices[0].message.content

def count_fields(header: str) -> int:
    """Count bit-position defines as a proxy for field count."""
    return len(re.findall(r'#define\s+\w+_POS', header))

def batch_generate(input_dir: str, output_file: str) -> None:
    inputs = sorted(pathlib.Path(input_dir).glob("*.txt"))
    if not inputs:
        print("No .txt files found.")
        return

    sections = []
    total_fields = 0

    for txt in inputs:
        print(f"Processing {txt.name}...")
        register_text = txt.read_text()
        header = generate_header(register_text)
        fields = count_fields(header)
        total_fields += fields
        sections.append(f"/* === {txt.stem} === */\n{header}\n")

    guard = output_file.upper().replace(".", "_").replace("/", "_")
    combined = (
        f"#ifndef {guard}\n#define {guard}\n\n"
        "#include <stdint.h>\n\n"
        + "\n".join(sections)
        + f"\n#endif /* {guard} */\n"
    )

    pathlib.Path(output_file).write_text(combined)
    print(f"\nSummary: {len(inputs)} registers, {total_fields} fields → {output_file}")

if __name__ == "__main__":
    import sys
    batch_generate(
        input_dir=sys.argv[1] if len(sys.argv) > 1 else "registers/",
        output_file=sys.argv[2] if len(sys.argv) > 2 else "output/peripherals.h"
    )
```

</details>
